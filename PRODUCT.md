# Product

## Register

brand

## Users

GroundK 팀 동료들(디자이너·개발자)이 다른 웹 프로젝트(groundk.com.kr 글로벌 사이트, T-RiseUp/RIDEUS 포탈, 각종 랜딩페이지)에서 제스처·애니메이션·재질 인터랙션을 구현할 때 참고하러 오는 내부 쇼케이스. 이미 자기 프로젝트가 있는 상태에서 "이런 느낌을 우리 사이트에도 넣고 싶다" 하고 둘러보는 맥락.

## Product Purpose

Emil Kowalski의 `apple-design` Claude Code 스킬(Apple WWDC 디자인 원칙)을 살아있는 인터랙션 데모로 번역해서 보여주고, 각 데모의 코드를 그대로 복사해 갈 수 있게 한다. 성공 기준: 팀원이 사이트를 보고 "이거 우리 프로젝트에 써야겠다"고 생각하며 실제로 코드를 복사해가는 것.

## Brand Personality

정교함(craft), 확신(unapologetic confidence), 인터랙션 자체가 주인공. 장난스럽거나 캐주얼하지 않고, 절제된 채로 강렬함.

## Anti-references

AI 슬롭처럼 보이는 모든 것 — 특히 impeccable 스킬의 절대 금지 목록(그라디언트 텍스트, 사이드 스트라이프 보더, 히어로+통계카드 SaaS 클리셰, 동일한 카드 그리드, 남용된 글래스모피즘). 뻔한 "AI가 만든 느낌"이 나면 실패.

## Design Principles

- 인터랙션이 콘텐츠다 — 셸의 비주얼 화려함은 데모 자체의 물리적 움직임을 돋보이게 하는 무대여야지, 데모와 경쟁하면 안 된다.
- 실천이 곧 증명이다(practice what you preach) — 사이트 자체의 전환·스크롤·타이포가 가르치는 원칙을 실제로 따라야 한다.
- 편집적 타이포그래피 우선, 장식은 최소. 큰 글자와 여백이 대부분의 작업을 하게 한다.
- 팀 참고용이므로 코드 재사용성(복사해서 바로 쓸 수 있음)이 시각적 화려함보다 항상 우선한다.

## Accessibility & Inclusion

`prefers-reduced-motion`을 사이트 자체가 실제로 지킨다(이미 6개 데모 중 하나가 이 원칙을 직접 다룸 — 셸 자체도 예외가 아니다). WCAG 특정 레벨 요구사항은 없음.
